3. 帧结构:从 SOF 到 EOF

CAN 2.0B 标准数据帧结构 交互

鼠标悬停在各字段上查看详细说明

DATA 可以是 0 字节吗?可以! DLC=0 的数据帧完全合法(ISO 11898-1 明确允许)。此时帧里 没有 DATA 段,DLC 字段后直接跟 CRC。用途:仅靠 CAN ID 本身就能传递"事件发生"的语义,例如 CANopen 中的 NMT 心跳帧、同步帧(SYNC)。帧头的 SOF/ID/DLC/CRC/ACK/EOF 照常存在,只是载荷为空。
ACK 机制是 CAN 协议的核心: 发送方在 ACK 位发隐性,期望任意接收方拉为显性。如果没人拉显性,发送方判定"没人 ACK",视为发送失败。

位填充 (Bit Stuffing)

CAN 使用 NRZ (Non-Return-to-Zero) 编码,不像 Manchester 编码那样自带时钟。接收方靠位边沿来同步自己的时钟。如果连续发太多相同极性的位,接收方会丢失同步。

解决方案:每 5 个连续同极性位后,强制插入一个反转位(填充位, stuff bit)。接收方看到填充位后丢弃它,不算入有效数据。

位填充模拟器 交互

输入一串原始位(0/1),观察填充位如何被插入。填充位用 红色 标记。

位填充对帧长的影响: 最坏情况下,每 4 个有效位就插入 1 个填充位(如 00000111110000011111...),帧长增加 25%。这意味着理论最大帧率要打折——计算总线负载时必须考虑填充位开销。
适用范围说明
SOF → CRC 段末尾填充位在此范围内插入(不含 CRC 界定符后的字段)
CRC_DEL / ACK / EOF不填充,这些字段是固定格式
CAN-FD 数据段CAN-FD 在数据段用固定填充(每 4 位插 1 位),不是传统的 5 位规则

错误帧 (Error Frame)

当节点检测到错误时,不是默默丢弃 — 它会主动向总线发送错误帧,强行破坏当前传输,通知所有节点"这一帧有问题"。

错误帧结构对比 交互

点击切换 Active / Passive 错误帧,观察区别

Active Error FramePassive Error Frame
发送条件TEC < 128 (Error Active 状态)TEC ≥ 128 (Error Passive 状态)
错误标志6 位显性(强行覆盖总线)6 位隐性(不干扰总线)
效果立即破坏当前帧,所有节点都知道出错只有自己知道出错,不影响其他节点
错误界定符8 位隐性(两种都一样)
为什么 6 位显性能破坏当前帧? 因为违反了位填充规则 — 正常数据最多 5 位同极性,6 位显性必然是非法序列。所有正在接收的节点都会检测到填充错误,丢弃当前帧。

Overload Frame(过载帧)也值得了解:节点在帧间隔期间如果还没准备好接收下一帧,可以发 6 位显性的过载标志来延迟总线。结构与 Active Error Frame 类似,但只在帧间隔(IFS)期间发送。

9. 双段速率:仲裁段 vs 数据段

BRS 速率切换时序图 动画

观察 CAN-FD 帧在仲裁段(1M)与数据段(5M)之间的速率切换

为什么仲裁段不能提速? 仲裁需要多节点同时驱动总线,信号往返时间必须 < bit 时间。40m 线缆往返 400ns,只能 1Mbps。
数据段为什么可以更快? 仲裁结束后只有获胜节点单独驱动,接收方被动监听,时序约束大幅放宽。

总线负载率计算

总线负载率 = 单位时间内总线被占用的时间 / 总时间。超过 ~70% 时低优先级帧的延迟急剧上升,超过 ~85% 系统基本不可用(高优先级帧饿死低优先级)。

经验法则: 每个电机每个控制周期至少 2 帧(1 命令 + 1 反馈)。MIT 模式每帧 ~130bit(含开销),1Mbps 下每帧占 130μs。4 电机 × 1kHz × 2 帧 × 130μs = 1040μs/ms = 104% → 超载! 解法:降频到 500Hz,或切 CAN-FD 5M。

实时调度 / 最坏响应时间 (WCRT)

多电机系统的核心问题:能否保证每个控制帧在截止时间内送达? CAN 的仲裁机制意味着低优先级帧可能被高优先级帧无限延迟(优先级反转)。

WCRT 公式(简化,单帧无抖动):

R_m = C_m + Σ_{j∈hp(m)} ⌈R_m / T_j⌉ × C_j + τ_block

其中 C_m=帧传输时间,hp(m)=比 m 优先级高的所有帧集合,T_j=帧 j 的周期,τ_block=被当前正在传输的低优先级帧阻塞的最大时间(= 最长帧的传输时间)。

设计建议:
  • 控制频率最高的电机分配最小 CAN ID(最高优先级)
  • WCRT 应 < 控制周期的 50%(留余量给反馈帧和抖动)
  • 如果 WCRT 超限 → 降低控制频率,或切到 CAN-FD 5M 缩短 C_m

CANopen 高层协议概览

裸 CAN 只定义了帧的传输(物理层+数据链路层),不定义帧里数据的含义。CANopen (CiA 301/402) 是最常用的 CAN 应用层协议,广泛用于工业伺服、机器人关节。

2.2 GND 共地

必须接 GND: 隔离电源(电机用独立 24V)的 GND 与适配器 GND 是两个不同电势参考。如果不接共地,差分电压可能超出 CAN 收发器(TJA1051)的 ±12V 共模范围 → 直接拒收。
解决: 适配器 GND ↔ 24V 电源负极 一根线连过去。

总线长度 vs 速率

CAN 的最大总线长度受限于信号往返延迟(仲裁段要求所有节点在同一 bit 内看到总线状态)和信号衰减。速率越高,bit 越短,允许的线缆越短。

CAN-FD 双速率的长度约束: 仲裁段 1M → 最长 40m;但数据段 5M → 最长约 5m。系统最大线长取决于数据段速率,所以 5M FD 模式下总线不能超过约 5 米。机器人关节间通常 <2m,完全没问题;但跨设备长距离连接就不行了。

Stub 长度限制

"Stub"是从主干线分支到节点的支线。支线末端是开路(节点的收发器输入阻抗极高 ≈ 开路),会产生反射。

Stub 反射原理
数据段速率最大 Stub 长度说明
1 Mbps (经典 CAN)< 30 cm (ISO 推荐)实测 1m 内通常能用,但不规范
2 Mbps (FD)< 15 cm开始严格
5 Mbps (FD)< 5 cm (几乎为零)节点必须直接在主干线上,不能有支线!
5M CAN-FD 的实际含义: Stub 长度限制 <5cm 意味着不允许使用引出线/跳线连接节点。正确做法:
  • 达妙电机的 XT30 串联方案(接口-1 进、接口-2 出)天然是菊花链(daisy-chain),Stub ≈ 0,完美
  • 如果自己设计 PCB,CAN 收发器必须直接焊在主干线上,不能用排线引出
  • T 型分支(star topology)在 5M 下基本不可用

CAN 收发器选型

CAN 收发器(transceiver)是MCU 数字逻辑和 CAN 总线之间的桥梁。MCU 的 CAN 控制器输出 TTL/CMOS 电平的 TX 信号,收发器将其转为差分信号驱动到总线上;反向地,收发器把总线上的差分信号转回单端 RX 给 MCU。

1CAN_H / CAN_L 差分对走线:等长、等间距、平行,间距 ≤ 2 倍线宽保证差模阻抗 ≈ 120Ω,减少 skew
2GND 铜皮连续完整,差分对下方不能有 GND 割缝割缝会改变回流路径,增加 EMI
3退耦电容紧贴收发器 VCC,走线 < 5mm吸收驱动总线时的瞬态大电流(可达 ±70mA)
4TVS 放最靠近连接器,在 PCB 边缘ESD 能量第一时间被钳位,不进入板内
5CAN 差分对远离高频数字线(SPI CLK、USB 等)避免串扰耦合
6终端电阻紧贴收发器,不要放在连接器端减少终端电阻和收发器之间的 stub 长度
75M CAN-FD:走线长度 < 5cm(收发器到连接器)高速信号对 stub 极度敏感

隔离设计

为什么需要隔离?当 CAN 节点之间的 GND 电势差较大(如不同电源域、不同机柜、有大电流地线压降),即使差分信号能承受 ±12V 共模,极端情况下仍可能损坏收发器。电气隔离在信号路径上"切断"电气连接,用磁耦合/光耦合传输信号。

18. MIT 模式

CTRL_MODE = 1, CAN ID 偏移 = 0x000(CAN ID 就是 ESC_ID 本身,无偏移)。

MIT 模式不走电机内部的位置环和速度环,而是由上位机每个控制周期发送 5 个参数,电机 MCU 用下面的公式直接算出目标力矩,送入最内层的电流环执行:

τ_cmd = Kp × (q_des − q_fdb) + Kd × (dq_des − dq_fdb) + τ_ff

这个公式运行在电机 MCU 内部,不是在上位机。上位机只负责把 5 个参数打包成 8 字节 CAN 帧发过去。电机 MCU 用自身编码器的实时 q_fdb / dq_fdb 代入公式,算出 τ_cmd,然后交给电流环(FOC)驱动三相桥。

达妙 MIT 模式控制框图 (DM-J4310 手册)

标准控制论框图。圆圈 = 求和节 (⊕ 加 / ⊖ 减),方块 = 传递函数/增益,箭头 = 信号流。虚线框内为电机 MCU 内部运算。

关键理解: MIT 公式里的 q_fdb / dq_fdb 是电机编码器的实时读数,在电机 MCU 内部获取,不是上位机发过去的。上位机只发 5 个参数(q_des, dq_des, Kp, Kd, τ_ff),电机自己完成位置/速度误差的计算和力矩输出。上位机通过反馈帧异步获得 q_fdb / dq_fdb / τ_fdb,用于下一帧的决策。

达妙 J4310 的 MIT 参数范围与量化

5 个参数在 CAN 帧里不是用 float 传输的,而是压缩成 12~16 bit 无符号整数,有精度损失:

参数物理范围量化位宽量化步长(精度)含义
q_des[-12.5, +12.5] rad16 bit≈ 0.00038 rad (0.022°)目标位置(输出轴)
dq_des[-30, +30] rad/s12 bit≈ 0.015 rad/s目标速度
Kp[0, 500]12 bit≈ 0.122位置刚度(弹簧系数)
Kd[0, 5]12 bit≈ 0.00122速度阻尼系数

PMAX / VMAX / TMAX 是可改的: 这三个映射范围存储在 RID=21/22/23 寄存器中,出厂默认 12.5/30/10。如果你的应用位置范围更小(比如 ±3 rad),可以把 PMAX 改成 3,同样的 16 bit 覆盖更小范围 → 精度提升 4 倍。改完需 save (0xAA) 持久化。代码侧的 encode_mit / decode_feedback 也要对应修改。

MIT 公式看起来是数学,但它描述的是一个物理弹簧-阻尼器系统。理解这个类比是掌握 MIT 模式的关键。

MIT 参数物理类比效果
Kp弹簧刚度 (N·m/rad)Kp 越大 → "弹簧越硬" → 偏离目标位置时回复力越强。Kp=0 → 没有弹簧,电机不会主动回到目标位置
Kd阻尼系数 (N·m·s/rad)Kd 越大 → "液体越粘稠" → 运动阻力越大。Kd=0 → 没有阻尼,系统会永远振荡(弹簧不会停)

dq_des阻尼器的参考速度阻尼力在此速度下为零;低于此速度阻尼反而"推",高于此速度阻尼"拉"
τ_ff外加恒力(如重力)不依赖位置/速度的恒定力矩,用于补偿已知外力
阻尼比 ζ 决定系统特性:
对于等效惯量 J 的电机:ζ = Kd / (2 × √(Kp × J))
  • ζ < 1 (欠阻尼) — 振荡衰减,响应快但有超调
  • ζ = 1 (临界阻尼) — 最快无超调回到目标,理想状态
  • ζ > 1 (过阻尼) — 无超调但回位慢,像在蜂蜜里移动
达妙 J4310 惯量 J ≈ 0.05 kg·m²(含减速器),Kp=50 时临界阻尼 Kd ≈ 2×√(50×0.05) ≈ 3.2

重力补偿:τ_ff 的核心用途

机械臂关节受重力影响,如果不补偿,弹簧(Kp)需要持续输出力矩来抵抗重力 → 位置会有稳态误差(弹簧被重力拉偏)。τ_ff 的主要用途就是前馈补偿已知的重力矩。

频率不够高时 Kp 调大会怎样?
离散控制器的稳定性条件(简化):Kp < 2·J / (Δt²),其中 Δt = 1/控制频率。
例如 J=0.05, 200Hz → Kp_max ≈ 2×0.05/(0.005²) = 4000 — 理论上限很高。
但实际中通信延迟、计算延迟会额外引入相位滞后,实际可用 Kp 远低于理论值。经验:1kHz 下 Kp 超过 200 开始出现颤振,500Hz 下 Kp 超过 100 需要小心。

MIT 没有 I 项意味着什么?
假设电机水平持位 Kp=50, q_des=0,受到恒定外力 F=1 N·m(如重力)。稳态时 Kp·(0−q_actual)=F → q_actual = −F/Kp = −0.02 rad。这个 0.02 rad 的偏差就是稳态误差,PID 的 I 项能通过积分消除它,但 MIT 没有 I 项。
MIT 的解法:用 τ_ff 补偿。设 τ_ff=F=1 N·m,则 Kp 只需处理残余误差,稳态误差趋近于 0。这比 I 项更好——I 项会引入相位滞后和积分饱和问题,τ_ff 没有这些副作用。

为什么需要 MIT 模式 — 与内部环路的本质区别

达妙内部已有位置环(KP_APR/KI_APR)和速度环(KP_ASR/KI_ASR),MIT 也做位置和速度反馈。区别不是功能,而是控制结构和权限分配。

串级 (POS_VEL)
# 发 p_des=2.0, v_max=5.0
dev.send(CAN_ID + 0x100,
  struct.pack('<ff', 2.0, 5.0))
# 电机自主走梯形曲线到 2 rad
# ...

# 碰到障碍物!
# 想变柔顺? 做不到。
# KP_APR 是寄存器值,
# 改它需要:
#   write_param(RID=27, new_kp)
#   save_params()  (flash!)
# 太慢了,而且有寿命限制
并联 (MIT)
# 高刚度趋近
dev.control_mit(CAN_ID,
  kp=80, kd=3, q=2.0, dq=0, tau=0)
# ...

# 碰到障碍物! 反馈帧显示 τ 突增
# 下一帧立即变柔顺:
dev.control_mit(CAN_ID,
  kp=5, kd=2, q=2.0, dq=0, tau=0)
#     ↑ 刚度降 16 倍,同一帧生效!
# 电机变成软弹簧,不会硬怼障碍物
什么时候串级更好?
  • 简单的点到点运动(不需要力控),POS_VEL 更省事——给目标位置就行,电机自己走梯形曲线
  • 通信不可靠时,POS_VEL 更安全——即使几帧丢失,电机内部环路仍在自主运行
  • 内部 PI 有 I 项,对恒定外力(如重力)有天然的稳态误差消除能力
什么时候并联(MIT)更好?
  • 需要控制接触力(力控、柔顺操作、人机交互)
  • 需要变刚度/变阻尼(不同阶段不同行为)
  • 需要高带宽轨迹跟踪(并联无串级延迟)
  • 需要前馈补偿(重力、惯性力、已知外力)
  • 多关节机器人——MIT 是行业标准接口(Mini Cheetah, Unitree A1, 小米 CyberDog 都用)
